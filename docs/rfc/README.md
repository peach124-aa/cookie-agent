# Requests for Comments (RFC) Directory

This directory stores Requests for Comments (RFCs), containing technical proposals for new subsystems, model pipelines, or major changes.

---

## Proposing RFCs

Use this directory to propose design implementations for review before starting code:

1.  **Drafting**: Write a detailed markdown proposal explaining the motivation, design options, execution flow, and testing strategy.
2.  **Naming Convention**: Save proposal files matching `docs/rfc/XXXX-short-title.md` (e.g. `docs/rfc/0002-vision-model-choice.md`).
3.  **Review**: Submit the RFC to the Tech Lead and team for review and approval.
